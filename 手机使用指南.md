# Org日记 Android应用 - 手机使用完整指南

## 📱 应用功能概览

本应用现已支持三大核心功能：
1. **添加条目** - 快速添加日记或TODO事项
2. **日记阅读** - 浏览最近的日记条目
3. **待办事项** - 查看所有TODO任务（类似Emacs Agenda）

---

## 🚀 首次安装与配置

### 第一步：安装APK

1. 将 `app/build/outputs/apk/debug/app-debug.apk` 传输到手机
2. 在手机上打开文件管理器，找到APK文件
3. 点击安装（可能需要允许"安装未知来源应用"）
4. 安装完成后，在桌面找到"Org日记"图标

### 第二步：获取GitHub Token

**重要：** 由于手机没有配置SSH密钥，应用使用GitHub Personal Access Token进行认证。

1. 在电脑上访问：https://github.com/settings/tokens
2. 点击 **"Generate new token (classic)"**
3. 设置Token名称（如：`org-diary-mobile`）
4. 勾选权限：**`repo`**（完整仓库访问权限）
5. 点击生成，**立即复制Token**（只显示一次！）
6. 将Token保存到安全的地方（建议使用密码管理器）

### 第三步：首次同步

1. 打开"Org日记"应用
2. 点击右上角 **"设置"** 按钮
3. 在"GitHub Token"输入框中粘贴刚才复制的Token
4. 点击 **"同步仓库"** 按钮
5. 等待同步完成（首次会克隆整个仓库，需要几秒到几十秒）
6. 看到"同步成功"提示后，配置完成！

## 📝 日常使用指南

### 界面导航

应用底部有三个标签页：
- **✏️ 添加** - 添加新的日记或TODO
- **📖 日记** - 查看最近的日记条目
- **📅 待办** - 查看所有TODO事项

---

## 功能一：添加日记条目

### 操作步骤

1. 确保在 **"添加"** 标签页
2. 点击 **"日记"** 按钮（默认选中）
3. 在"条目内容"输入框中输入日记内容
4. 点击 **"插入条目"** 按钮
5. 等待"插入成功"提示

### 生成格式示例

```org
* <2026-04-03 四>
**** <2026-04-03 四 11:30>
今天天气不错，完成了项目的重要功能。
```

### 注意事项

- 时间戳自动使用当前时间
- 内容会自动插入到main.org对应日期下
- 插入后自动推送到GitHub

---

## 功能二：添加TODO事项

### 操作步骤

1. 在 **"添加"** 标签页
2. 点击 **"TODO"** 按钮切换模式
3. 在"条目内容"输入框中输入TODO任务描述
4. **（可选）** 点击 **"📅 选择计划时间"** 按钮
   - 弹出日历选择器
   - 滑动选择年月日
   - 点击"确定"保存，或"取消"放弃
   - 已选择的日期会显示在按钮上
   - 点击"清除计划时间"可删除已选日期
5. **（可选）** 点击 **"⏰ 选择截止时间"** 按钮
   - 操作方式同计划时间
6. 点击 **"插入条目"** 按钮

### 生成格式示例

**带计划和截止时间：**
```org
* <2026-04-03 四>
**** TODO <2026-04-03 四 11:30>
SCHEDULED: <2026-04-05 六> DEADLINE: <2026-04-10 四>
完成项目文档编写
```

**仅TODO（无时间）：**
```org
* <2026-04-03 四>
**** TODO <2026-04-03 四 11:30>
购买生活用品
```

### 时间字段说明

- **计划时间（SCHEDULED）**：打算什么时候开始做这件事
- **截止时间（DEADLINE）**：必须在什么时候之前完成
- 两个字段都是可选的，可以只填一个或都不填

---

## 功能三：查看日记

### 操作步骤

1. 点击底部 **"📖 日记"** 标签
2. 自动加载最近50条日记条目
3. 每条卡片显示：
   - 时间戳（蓝色）
   - TODO标记（如果是TODO条目，显示红色"TODO"）
   - 条目内容
4. 点击右上角 **"刷新"** 按钮可重新加载

### 界面特点

- **卡片式设计**：每条日记独立显示，清晰易读
- **时间优先**：按时间倒序排列，最新的在最上面
- **滚动浏览**：支持上下滑动查看更多条目
- **自动适配**：针对手机屏幕优化，字体大小适中

---

## 功能四：查看待办事项（Agenda）

### 操作步骤

1. 点击底部 **"📅 待办"** 标签
2. 自动加载所有TODO事项
3. 每个TODO卡片显示：
   - 任务内容（粗体）
   - 创建时间
   - 📅 计划时间（如果有）
   - ⏰ 截止时间（如果有）
4. 点击右上角 **"刷新"** 按钮可重新加载

### 界面特点

- **彩色卡片**：TODO使用特殊背景色，与日记区分
- **时间提醒**：计划和截止时间用不同颜色标注
  - 计划时间：蓝色 📅
  - 截止时间：红色 ⏰
- **一目了然**：所有待办事项集中显示，方便管理

## ⚙️ 设置功能

### 何时需要打开设置

- 首次使用时配置Token
- Token过期需要更新
- 需要手动同步仓库

### 如何使用设置

1. 点击右上角 **"设置"** 按钮
2. 输入或更新GitHub Token
3. 点击 **"同步仓库"** 执行同步
4. 同步成功后，设置界面自动关闭

### 同步说明

- **首次同步**：克隆完整仓库（较慢）
- **后续同步**：只拉取更新（很快）
- **自动同步**：每次添加条目后自动推送
- **手动同步**：在设置中可随时手动同步

## 💡 使用技巧

### 1. 快速记录

- 打开应用默认在"添加"页面，可立即输入
- 输入完成后直接点击"插入条目"，无需其他操作

### 2. 查看历史

- 在"日记"标签可快速浏览最近记录
- 支持滚动查看更多历史条目

### 3. 管理TODO

- 在"待办"标签查看所有未完成任务
- 带截止时间的任务会用红色标注，提醒注意

### 4. 多设备协同

```
场景1：外出记录
手机添加 → 自动推送到GitHub → 回家后电脑git pull即可看到

场景2：电脑编辑
电脑修改main.org → git push → 手机打开应用点击刷新 → 看到最新内容
```

### 5. Token安全

- Token相当于密码，不要分享给他人
- 建议定期更换Token（在GitHub设置中）
- 如果Token泄露，立即在GitHub删除旧Token并生成新的

## 🔧 常见问题

### Q1: 同步失败怎么办？

**可能原因：**
- 网络连接问题 → 检查手机网络
- Token无效或过期 → 重新生成Token
- 仓库地址错误 → 确认是 `https://github.com/tacmon/my-diary-repo.git`

**解决方法：**
1. 检查网络连接
2. 在GitHub重新生成Token
3. 在设置中更新Token并重新同步

### Q2: 插入条目失败？

**可能原因：**
- 未完成首次同步 → 先在设置中同步仓库
- main.org文件不存在 → 确保仓库中有main.org
- Token权限不足 → 确保Token有`repo`权限

### Q3: 看不到日记或TODO？

**解决方法：**
1. 点击"刷新"按钮
2. 确认main.org中确实有对应内容
3. 检查条目格式是否正确

### Q4: 如何选择日期？

- 点击"📅 选择计划时间"或"⏰ 选择截止时间"按钮
- 在弹出的日历中滑动选择年、月、日
- 点击"确定"保存选择
- 如需清除已选日期，点击下方的"清除"按钮
- **无需手动输入**，所有日期通过日历选择器完成

### Q5: 如何在电脑上查看手机添加的内容？

```bash
# 在电脑上执行
cd ~/文档/每日工作记录/my-diary-repo
git pull
emacs main.org  # 或用其他编辑器打开
```

## 📊 状态提示说明

应用底部会显示当前操作状态：

- **"就绪"** - 可以正常使用
- **"同步中..."** - 正在同步仓库，请等待
- **"同步成功"** - 仓库同步完成
- **"同步失败: xxx"** - 同步出错，查看错误信息
- **"插入中..."** - 正在添加条目
- **"插入成功"** - 条目已添加并同步到GitHub
- **"插入失败"** - 添加失败，检查网络和配置
- **"推送失败: xxx"** - 条目已添加但推送失败，查看错误信息
