# Org日记 Android应用使用指南

## 项目概述

这是一个原生Android应用，用于在手机上快速添加条目到main.org日记文件，并通过GitHub仓库实现多设备同步。

## 功能特性

- ✅ 快速添加带时间戳的条目
- ✅ 自动使用当前时间
- ✅ 直接插入到main.org文件
- ✅ 与GitHub仓库同步
- ✅ 简洁的Material Design 3 UI

## 构建APK

### 前置要求

1. 安装Android Studio或Android SDK
2. 安装JDK 8或更高版本
3. 确保已配置ANDROID_HOME环境变量

### 构建步骤

```bash
cd 移动端实现/tri_2_apk

# 首次构建需要下载Gradle wrapper
./gradlew wrapper --gradle-version=8.2

# 构建Debug版本APK
./gradlew assembleDebug
```

生成的APK位于：`app/build/outputs/apk/debug/app-debug.apk`

## 安装到手机

### 方法1：通过USB连接

```bash
# 确保手机已开启USB调试
adb install app/build/outputs/apk/debug/app-debug.apk
```

### 方法2：直接传输

1. 将APK文件传输到手机（通过USB、蓝牙、云盘等）
2. 在手机上打开文件管理器
3. 点击APK文件
4. 允许安装未知来源应用
5. 完成安装

## 首次配置

### 1. 创建GitHub Personal Access Token

1. 访问 https://github.com/settings/tokens
2. 点击 "Generate new token (classic)"
3. 勾选 `repo` 权限（完整仓库访问）
4. 生成并复制token（注意保存，只显示一次）

### 2. 配置应用

1. 打开"Org日记"应用
2. 点击右上角"设置"按钮
3. 粘贴GitHub Token
4. 点击"同步仓库"按钮
5. 等待同步完成（首次会克隆整个仓库）

## 日常使用

### 添加条目

1. 在主界面输入条目内容
2. 点击"插入条目"按钮
3. 应用会自动：
   - 使用当前时间作为时间戳
   - 插入到main.org对应日期下
   - 提交并推送到GitHub

### 查看状态

界面底部会显示当前操作状态：
- "就绪" - 可以添加条目
- "插入中..." - 正在处理
- "插入成功" - 条目已添加并同步
- "同步失败" - 检查网络和token

## 工作流程

### 场景1：外出时快速记录

```
手机 → 打开应用 → 输入内容 → 点击插入 → 自动同步到GitHub
```

### 场景2：多设备协同

```
电脑编辑 → git push → 手机打开应用前会自动pull → 添加条目 → 自动push
```

## 技术实现

### 核心组件

1. **GitRepository.kt** - Git操作封装
   - 使用JGit库实现Git功能
   - 支持clone、pull、commit、push

2. **OrgFileManager.kt** - Org文件解析和编辑
   - 自动查找或创建日期标题
   - 按时间戳插入条目
   - 保持org-mode格式

3. **MainActivity.kt** - UI界面
   - Jetpack Compose实现
   - Material Design 3风格
   - 协程处理异步操作

### 条目格式

应用会自动生成如下格式的条目：

```org
* <2026-04-02 四>
**** <2026-04-02 四 22:30>
这是条目内容
```

## 注意事项

1. **Token安全**：不要分享你的GitHub Token
2. **网络要求**：同步需要网络连接
3. **同步习惯**：建议每次使用前先同步，避免冲突
4. **存储空间**：应用会在内部存储克隆完整仓库

## 故障排除

### 同步失败

- 检查网络连接
- 确认Token有效且有repo权限
- 查看GitHub仓库是否可访问

### 插入失败

- 确保已完成首次同步
- 检查main.org文件是否存在
- 查看状态栏错误信息

### 构建失败

- 确认Android SDK已正确安装
- 检查ANDROID_HOME环境变量
- 尝试清理：`./gradlew clean`
